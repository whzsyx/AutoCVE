# AuditAI UI Redesign Design

**Date:** 2026-03-27

**Goal**

将当前 `DeepAudit` 前端改造成浅灰系、轻商务 + 高效率工作台风格的 `AuditAI` 品牌界面，弱化原项目“赛博终端”和二开感，同时保留现有核心功能入口与业务流程。

## Design Direction

本次设计采用 `A + C` 混合方向：

- `A` 负责整体视觉语气：浅灰底色、白色浮层、柔和圆角、图 1 式左侧功能栏、正常无衬线字体。
- `C` 负责核心工作台效率：在首页、项目页、审计页提升信息分区和操作密度，让页面更像成熟 SaaS 产品而不是展示页。

整体视觉关键词：

- Soft executive
- Neutral gray
- Calm productivity
- Lightweight depth
- Clean Chinese sans-serif typography

## Scope

本次改版包含：

- 全局主题变量、背景、阴影、圆角、表单/弹窗/卡片语义重做
- 全局字体替换，去除像素风/大量等宽字体的默认使用
- 应用壳层与左侧导航栏重做
- `Dashboard`、`Projects`、`AgentAudit` 三个核心页面重排
- 用户可见品牌文案从 `DeepAudit` 统一替换为 `AuditAI`
- Logo / favicon / 登录注册页与侧栏图标替换为“小狗”方案

本次不主动重构业务 API 或扫描流程逻辑，只调整 UI、信息架构和用户可见品牌层。

## Information Architecture

### Global Shell

- 左侧固定导航：白色卡片式侧栏，顶部为狗头 logo + `AuditAI`
- 中部为主要功能导航，激活项使用低饱和浅绿色/灰绿色高亮
- 底部为账户入口和版本信息
- 主内容区使用浅灰渐变背景，内容容器统一采用大圆角白卡

### Dashboard

- 顶部欢迎区 + 搜索/状态信息
- 第一屏显示关键指标卡
- 中部为近期任务、项目概览、质量趋势等内容
- 右侧保留快捷操作与系统概况，但去掉“赛博终端”表达

### Projects

- 顶部工具条：标题、搜索、筛选、创建项目入口
- 主体为更现代的项目卡片/列表工作台
- 创建、编辑、删除弹窗改成正常产品弹窗，不再模拟终端窗口
- ZIP / 仓库项目差异仍保留，但通过更清晰的表单分组表达

### Agent Audit

- 采用高效率三段式工作区
- 左侧为活动日志
- 中间为当前任务状态、阶段进度与主要操作
- 右侧为 Agent 树、任务统计和导出入口
- 保留现有实时感，但去掉“终端”“发光”“像素”视觉语言

## Brand Replacement

所有用户可见品牌入口替换为 `AuditAI`，重点包括：

- 页面标题和 meta
- 侧栏 / 登录 / 注册 / 404 / 报告导出
- Splash 文案、欢迎语、站点名
- 本地存储 key / 默认下载文件名中对品牌敏感的标识

不强制修改所有底层包名或历史文档，但会优先处理运行时、UI 层和用户可见输出。

## Typography

- 主字体统一为正常中文无衬线字体栈
- 标题与正文不再默认使用 `font-mono`
- 仅在少量日志、代码、终端输出区域保留等宽字体

推荐字体栈：

- `"PingFang SC", "Microsoft YaHei", "Noto Sans SC", "Helvetica Neue", Arial, sans-serif`

## Color System

主色方案：

- 背景：雾灰、暖灰、米白
- 卡片：高亮白
- 边框：低对比度暖灰
- 强调色：低饱和鼠尾草绿 / 冷灰绿
- 次强调：浅雾蓝，仅在图表和状态辅助使用

禁用方向：

- 赛博橙黑
- 大面积终端绿
- 像素字体主导
- 强发光、扫描线和终端窗口装饰

## Assets

- 使用你提供的狗头图标作为主品牌图形
- 去掉外部圆圈，只保留狗头主体
- 同步生成 / 替换：
  - favicon
  - 侧栏 logo
  - 登录注册页 logo
  - 可能的头图与品牌展示位置

## Error Handling

- 保持现有错误处理逻辑不变
- 仅调整错误状态页和 toast 的视觉风格
- 如果局部页面改版引发样式冲突，优先通过局部容器样式隔离解决，而不是回退全局主题

## Testing Strategy

- 先建立前端回归测试基线：执行 `type-check` / `build`
- 改版完成后再次执行 `type-check` / `build`
- 因现有项目前端未见现成页面级测试，本次以类型检查和打包验证为主
- 对关键品牌替换和页面结构通过代码搜索确认无遗漏

## Constraints And Notes

- 当前工作区不是 git 仓库，无法按技能要求提交设计文档 commit
- 由于当前环境的可视化辅助浏览器服务不可稳定访问，本次设计确认通过终端文本完成
- 不大规模改动非核心页面布局，但会同步修正其中明显的品牌文案
